Website Redesign Implementation Guide
Overview
This guide provides a systematic approach to redesigning your existing website using the Emma Smith portfolio design as a reference. Follow these phases to ensure a smooth transition from your current design to the new modern, glassmorphic aesthetic.

Phase 1: Analysis & Planning
Step 1.1: Audit Your Current Website
Objective: Understand your existing codebase and structure.

Instructions for Codex:

Analyze the existing website code and provide a comprehensive report including:

1. **File Structure Analysis**
   - List all HTML, CSS, and JavaScript files
   - Identify the main entry point (index.html or equivalent)
   - Map out the folder/directory structure
   - Note any frameworks or libraries being used (React, Vue, Bootstrap, etc.)

2. **Layout Analysis**
   - Identify the current layout structure (header, main, footer, sections)
   - Document the navigation structure and menu items
   - List all page sections and their purposes
   - Note responsive breakpoints and mobile behavior

3. **Content Inventory**
   - Extract all text content (headings, paragraphs, buttons)
   - List all images and their purposes
   - Document all links and their destinations
   - Identify forms and interactive elements

4. **Styling Analysis**
   - Identify the CSS methodology (inline, external, CSS-in-JS)
   - Document color scheme (primary, secondary, accent colors)
   - List typography (fonts, sizes, weights)
   - Note spacing patterns (margins, padding)
   - Identify animations and transitions

5. **Functionality Analysis**
   - List all interactive features (buttons, forms, modals)
   - Document JavaScript functionality
   - Note any third-party integrations (analytics, APIs)
   - Identify event listeners and user interactions

document.querySelectorAll('a[href^="#"]').forEach(anchor => {

## Phase 7: Performance Optimization

### Step 7.1: Asset Optimization
**Objective:** Optimize all assets for fast loading.

**Instructions for Codex:**
Optimize the following:

Images

Compress images (TinyPNG, ImageOptim)

Use WebP format with fallbacks

Implement lazy loading

Use appropriate dimensions

Add width/height attributes to prevent layout shift

<picture>
  <source srcset="profile.webp" type="image/webp">
  <img src="profile.jpg" alt="Profile" loading="lazy" width="420" height="420">
</picture>
CSS

Minify CSS files

Remove unused styles

Combine multiple CSS files

Use critical CSS for above-the-fold content

JavaScript

Minify JS files

Defer non-critical scripts

Use async loading where appropriate

Remove console.logs and comments

Fonts

Use font-display: swap

Preload critical fonts

Subset fonts to include only needed characters

<link rel="preconnect" href="https://fonts.googleapis.com">
<link rel="preconnect" href="https://fonts.gstatic.com" crossorigin>
<link href="https://fonts.googleapis.com/css2?family=Poppins:wght@300;400;500;600;700&display=swap" rel="stylesheet">
